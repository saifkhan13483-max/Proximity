I need you to audit and refine the color palette for Proximity Credit Repair — a high-end credit repair marketing site and client portal built with React 18 + Vite + Tailwind CSS v3 + shadcn/ui + Framer Motion.

Current Design Philosophy (preserve this)

The brand is built around a "gold on dark" luxury aesthetic:

Dark layered backgrounds: #0A0A0A → #141414 → #1A1A1A
Signature gold #B8924A appears on every interactive element
Warm off-white #F9F6F1 used instead of pure white to maintain warmth
Diagonal gold gradient (#B8924A → #D4AF72 → #8B6A2E at 135deg) across buttons, text highlights, dividers
The Problem to Solve

The current palette looks premium but has real-world readability and accessibility issues:

Gold #B8924A on dark backgrounds may fail WCAG AA contrast (4.5:1 for body text, 3:1 for large text/UI).
Muted text #6B6B6B on white likely sits below AA contrast.
Body text #1A1A1A on #F9F6F1 is fine, but on #141414 cards it would be invisible — clarify which surface owns which text token.
Gold gradients can lose legibility when used on text (especially small text and on light surfaces).
There's no clearly defined semantic state palette (success, warning, info, destructive-on-dark) that harmonizes with gold.
Hover, active, focus, and disabled states aren't explicitly defined — they're often the weakest link in luxury sites.
The dark-mode token set (for /dashboard and /admin) isn't fully specified — only light-mode shadcn tokens are listed.
